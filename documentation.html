SplitBuddy

Developer Documentation & Comprehensive Interview Guide

📦 Project Overview

SplitBuddy is a high-performance MERN-stack inspired application designed for effortless expense management among friends and groups.

Frontend Focus

Built with Vue 3 & Pinia, leveraging the Composition API for highly modular and reactive UI components.

Robust Backend

Powered by Node.js/Express and MongoDB, handling complex financial calculations and data persistence.

Technology Stack Breakdown

  • Vue 3: Reactive framework using script setup syntax.
  • Pinia: Modern state management (Store/Actions/Getters).
  • Tailwind CSS: Utility-first styling for premium responsive design.
  • JWT Auth: Stateless authentication with encrypted tokens.
  • Mongoose: Schema-based modeling for data integrity.

🎨 Frontend Architecture

The frontend is architected as a modern Single Page Application (SPA) prioritizing performance and developer experience.

src/
├── services/     # Axios interceptors & API calls
├── store/       # Global state (Auth, User data)
├── pages/       # Routed views (Auth, Dashboard)
└── components/   # Reusable UI (Modals, Charts)

Key Architectural Wins:

  • Axios Interceptors: Automatically handles token injection and global 401 error redirects.
  • Navigation Guards: Route protection logic to ensure guest-only and auth-only views.
  • Responsive Grid: Used Tailwind flexbox and grid systems to ensure mobile-tablet-desktop parity.
  • State Hydration: Automatic session recovery from localStorage on app boot.

⚙️ Core Features & Logic

Complex Implementation

1. The Split Engine

Implementing the splitting logic required three distinct mathematical models in the frontend:

  • Equal Split: Amount / Participants with precision rounding.
  • Exact Amount: Real-time sum validation using computed properties.
  • Percentage Split: Ensure Σ (%) = 100 before allowing form submission.
Dual Authentication

2. Modern Auth Flow

A unified authentication experience supporting both legacy and modern social logins:

  • Standard Auth: Manual Email/Password with client-side hashing support.
  • Google One-Tap: Integration with accounts.google.com/gsi for effortless entry.
  • Consolidated UI: A single, reactive page handling Signup, Login, and Social Auth in one view.

🎙️ Interview Preperation

Use these specific technical answers when asked by an interviewer.

Q: How did you handle state management?

A: "I chose Pinia for SplitBuddy because it offers a clean, modular architecture that integrates seamlessly with Vue 3's Composition API. I used it to manage user sessions globally so that components could react to authentication changes without prop drilling."

Q: How did you ensure accuracy in splitting?

A: "Accuracy was a priority. I built a validation layer using Vue watchers and computed properties that verify total sums and percentages in real-time. This prevents users from submitting invalid financial data before it even hits the backend."

Q: Tell us about the premium UI design.

A: "I implemented a premium design system using Tailwind CSS, focusing on glassmorphism and smooth transitions. I used backdrop filters and deep shadows to create a sense of depth, ensuring the app feels more like a modern native application."
